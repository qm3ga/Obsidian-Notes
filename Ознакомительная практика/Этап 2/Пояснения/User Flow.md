# Что это
User flow это визуальная блок-схема, описывающая пошаговый путь пользователя по сайту или приложению для достижения конкретной цели

# Участник 1 — Вход и ориентация

(Главная, О кафедре, Контакты)

---

## User Flow 1 — Первичное знакомство с платформой

### Логика:

Начало → Главная → Просмотр контента → Выбор действия → Переход в каталог → Конец

### Описание:

Пользователь впервые заходит на сайт, знакомится с платформой через главную страницу и переходит в каталог работ для дальнейшего просмотра.

---

## User Flow 2 — Поиск информации о кафедре

### Логика:

Начало → Главная → Меню → О кафедре → Просмотр информации → Контакты → Конец

### Описание:

Пользователь (например, абитуриент) переходит в раздел «О кафедре», изучает информацию и при необходимости обращается к контактам.

---

# Участник 2 — Навигация и поиск

(Каталог, Поиск, Подборки)

---

## User Flow 3 — Просмотр работ без регистрации

### Логика:

Начало → Главная → Каталог → Просмотр карточек → Выбор работы → Страница проекта → Конец

### Описание:

Пользователь переходит в каталог, просматривает список работ и открывает интересующий проект.

---

## User Flow 4 — Поиск конкретной работы

### Логика:

Начало → Главная → Поиск → Ввод запроса → Результаты → Выбор работы → Страница проекта → Конец

### Описание:

Пользователь использует поиск для нахождения конкретной работы и переходит к её просмотру.

---

# Участник 3 — Просмотр контента

(Страница проекта, Похожие работы, Пустые состояния)

---

## User Flow 5 — Детальный просмотр работы

### Логика:

Начало → Каталог → Страница проекта → Просмотр галереи → Чтение описания → Конец

### Описание:

Пользователь изучает содержимое проекта: изображения, описание и дополнительные материалы.

---

## User Flow 6 — Переход к похожим работам

### Логика:

Начало → Страница проекта → Блок "Похожие работы" → Выбор работы → Новая страница проекта → Конец

### Описание:

После просмотра одной работы пользователь переходит к похожим, продолжая исследование контента.

---

# Участник 4 — Пользовательская система

(Профиль, Добавление работы, Авторизация)

---

## User Flow 7 — Регистрация пользователя

### Логика:

Начало → Главная → Авторизация → Регистрация → Ввод данных → Подтверждение → Профиль → Конец

### Описание:

Пользователь проходит процесс регистрации и получает доступ к личному профилю.

---

## User Flow 8 — Добавление новой работы

### Логика:

Начало → Авторизация → Вход → Профиль → Добавить работу → Заполнение формы → Загрузка → Публикация → Конец

### Описание:

Авторизованный пользователь добавляет новую работу в систему через форму загрузки
